## 1. 프론트엔드 작업

### 1.1 카카오맵 실사용 컴포넌트 연결

기존 인프라 브리핑 화면은 목업 지도 중심으로 동작하고 있었고, 카카오맵 SDK를 실제로 붙이기 위한 구조가 필요했다.

추가/수정한 파일:

- `frontend/src/features/map/InfraMap.jsx`
- `frontend/src/features/map/kakaoLoader.js`
- `frontend/src/legacy/infra.jsx`
- `frontend/index.html`
- `frontend/.env.example`

주요 내용:

- `kakaoLoader.js`에서 카카오 JavaScript SDK를 동적으로 로드하도록 구성했다.
- 카카오맵 JavaScript 키는 코드에 직접 넣지 않고 `VITE_KAKAO_MAP_KEY` 환경변수로 읽는다.
- `InfraMap.jsx`에서 주소 기준 좌표 변환, 지도 생성, 주변 시설 검색, 마커 표시 흐름을 담당한다.
- 검색 필터는 대중교통, 병원/약국, 마트/편의점, 반려동물 관련 장소 중심으로 구성했다.
- 기존 `legacy/infra.jsx`에서는 `t.mapSrc === "mock"`일 때만 목업 지도를 쓰고, 그 외에는 새 `LiveInfraMap`을 렌더링하도록 연결했다.
- `index.html`에서 Vite 진입점은 `/src/main.jsx` 하나만 사용하도록 정리했다. 카카오 SDK는 HTML에 직접 넣지 않고 loader에서 관리한다.

로컬 환경변수 예시:

```properties
VITE_KAKAO_MAP_KEY=카카오_JavaScript_키
```

카카오 개발자 콘솔에서 필요한 도메인:

```text
http://localhost:5173
http://127.0.0.1:5173
```

### 1.2 카카오 OAuth 콜백 화면 추가

카카오 로그인 성공 후 백엔드가 프론트로 JWT를 전달할 수 있도록 `/oauth/callback` 진입점을 추가했다.

추가/수정한 파일:

- `frontend/src/features/member/oauth.js`
- `frontend/src/features/member/OAuthCallback.jsx`
- `frontend/src/shared/api/authFetch.js`
- `frontend/src/main.jsx`

주요 내용:

- `oauth.js`
  - 카카오 로그인 시작 URL을 `http://localhost:8080/oauth2/authorization/kakao`로 이동시키는 helper를 추가했다.
  - JWT 저장 키는 `zipt_access_token`으로 정리했다.
  - 로그인/로그아웃/토큰 조회 helper를 분리했다.
- `OAuthCallback.jsx`
  - 백엔드에서 전달한 `?token=` 값을 읽어 localStorage에 저장한다.
  - 저장 후 `/`로 이동한다.
  - OAuth 실패 시 `?error=&message=` 값을 화면에 보여주도록 수정했다.
- `authFetch.js`
  - API 요청 시 localStorage의 JWT를 `Authorization: Bearer ...` 헤더로 붙인다.
  - 401 응답이 오면 토큰을 제거하고 로그아웃 상태로 처리한다.
- `main.jsx`
  - `window.location.pathname === "/oauth/callback"`일 때는 레거시 앱 대신 `OAuthCallback`만 렌더링한다.

백엔드 origin 기본값:

```text
http://localhost:8080
```

필요하면 프론트 환경변수로 변경 가능하다.

```properties
VITE_BACKEND_ORIGIN=http://localhost:8080
```

### 2.2 회원 도메인 추가

추가한 파일:

- `backend/src/main/java/com/zipt/member/domain/Member.java`
- `backend/src/main/java/com/zipt/member/repository/MemberRepository.java`
- `backend/src/main/java/com/zipt/member/controller/MemberController.java`
- `backend/src/main/resources/db/migration/V2__create_members.sql`

회원 테이블 주요 컬럼:

- `id`
- `provider`
- `provider_id`
- `name`
- `email`
- `role`
- `created_at`
- `updated_at`

제약 조건:

- `provider + provider_id` 조합 unique

현재 카카오 로그인 기준:

- `provider`: `kakao`
- `providerId`: 카카오 사용자 id
- `name`: 카카오 닉네임
- `email`: 동의항목 권한이 있을 때만 저장 가능
- `role`: 기본 `USER`

회원 API:

```http
GET /api/v1/members/me
Authorization: Bearer {accessToken}
```

응답 예시:

```json
{
  "id": 1,
  "name": "카카오닉네임",
  "email": null,
  "role": "USER"
}
```

### 2.3 카카오 OAuth2 로그인 처리

추가한 파일:

- `backend/src/main/java/com/zipt/member/service/KakaoOAuth2UserService.java`
- `backend/src/main/java/com/zipt/global/security/SecurityConfig.java`
- `backend/src/main/java/com/zipt/global/security/OAuth2SuccessHandler.java`
- `backend/src/main/java/com/zipt/global/security/OAuth2FailureHandler.java`

로그인 흐름:

1. 프론트 또는 브라우저에서 아래 주소로 진입한다.

```text
http://localhost:8080/oauth2/authorization/kakao
```

2. Spring Security가 카카오 인증 페이지로 리다이렉트한다.
3. 사용자가 카카오 로그인 및 동의를 완료한다.
4. 카카오가 아래 백엔드 콜백으로 돌아온다.

```text
http://localhost:8080/login/oauth2/code/kakao
```

5. 백엔드가 카카오 Access Token을 발급받는다.
6. 백엔드가 카카오 사용자 정보 API를 호출한다.

```text
https://kapi.kakao.com/v2/user/me
```

7. `KakaoOAuth2UserService`가 회원을 조회하거나 새로 저장한다.
8. `OAuth2SuccessHandler`가 자체 JWT를 발급한다.
9. 프론트 콜백으로 이동한다.

```text
http://localhost:5173/oauth/callback?token={JWT}
```

10. 프론트가 JWT를 localStorage에 저장한다.

### 2.4 JWT 인증 필터 추가

추가한 파일:

- `backend/src/main/java/com/zipt/global/security/JwtTokenProvider.java`
- `backend/src/main/java/com/zipt/global/security/JwtAuthenticationFilter.java`

주요 내용:

- 로그인 성공 시 `memberId`와 `name` claim을 담은 JWT를 발급한다.
- API 요청의 `Authorization` 헤더에서 Bearer 토큰을 읽는다.
- 토큰이 유효하면 Spring Security `Authentication`에 `memberId`를 principal로 넣는다.

토큰 설정:

```properties
app.jwt.secret=${JWT_SECRET:change-me-to-a-long-random-32byte-min-secret}
app.jwt.expiration-ms=${JWT_EXPIRATION_MS:3600000}
```

주의:

- 운영/배포 환경에서는 `JWT_SECRET`을 반드시 충분히 긴 랜덤 문자열로 교체해야 한다.
- 현재 기본값은 로컬 개발용이다.

### 2.5 백엔드 설정 파일 정리

수정한 파일:

- `backend/src/main/resources/application.properties`
- `backend/.gitignore`
- `backend/.env`

`application.properties` 주요 추가 설정:

```properties
spring.config.import=optional:file:.env[.properties]

app.oauth2-redirect-uri=${FRONTEND_OAUTH_CALLBACK:http://localhost:5173/oauth/callback}

spring.security.oauth2.client.registration.kakao.client-id=${KAKAO_CLIENT_ID:local-kakao-client-id}
spring.security.oauth2.client.registration.kakao.client-secret=${KAKAO_CLIENT_SECRET:}
spring.security.oauth2.client.registration.kakao.client-authentication-method=${KAKAO_CLIENT_AUTH_METHOD:none}
spring.security.oauth2.client.registration.kakao.authorization-grant-type=authorization_code
spring.security.oauth2.client.registration.kakao.redirect-uri=${KAKAO_REDIRECT_URI:http://localhost:8080/login/oauth2/code/kakao}
spring.security.oauth2.client.registration.kakao.scope=profile_nickname
spring.security.oauth2.client.provider.kakao.authorization-uri=https://kauth.kakao.com/oauth/authorize
spring.security.oauth2.client.provider.kakao.token-uri=https://kauth.kakao.com/oauth/token
spring.security.oauth2.client.provider.kakao.user-info-uri=https://kapi.kakao.com/v2/user/me
spring.security.oauth2.client.provider.kakao.user-name-attribute=id
```

`.env`는 로컬 전용 파일로 사용한다.

```properties
KAKAO_CLIENT_ID=카카오_REST_API_키
KAKAO_CLIENT_SECRET=카카오_Client_Secret
KAKAO_CLIENT_AUTH_METHOD=client_secret_post
```

`.env`는 `backend/.gitignore`에 추가했기 때문에 원격 저장소에 올라가지 않는다.

```

```

### 3.2 카카오 로그인

사용 키:

- REST API 키

등록 위치:

```text
제품 설정 → 카카오 로그인 → 일반 → 카카오 로그인 리다이렉트 URI
```

등록값:

```text
http://localhost:8080/login/oauth2/code/kakao
```

동의항목:

- `profile_nickname`: 필수 동의
- `profile_image`: 현재 필수 아님
- `account_email`: 현재 권한 없음. 이메일이 필요해지면 추가 권한 신청 필요

동의 목적 예시:

```text
카카오 로그인 회원 식별 및 마이페이지 사용자 정보 표시
```

OpenID Connect:

- 현재 구조에서는 OFF로 둔다.
- 지금 코드는 ID Token 기반 로그인이 아니라 OAuth2 사용자 정보 API 기반 로그인이다.

Client Secret:

- REST API 키 카드 아래의 `클라이언트 시크릿` 버튼에서 생성/활성화했다.
- 활성화 후 `.env`에 `KAKAO_CLIENT_SECRET`, `KAKAO_CLIENT_AUTH_METHOD=client_secret_post`를 추가했다.

### 1.1 카카오맵 실사용 컴포넌트 연결

기존 인프라 브리핑 화면은 목업 지도 중심으로 동작하고 있었고, 카카오맵 SDK를 실제로 붙이기 위한 구조가 필요했다.

추가/수정한 파일:

- `frontend/src/features/map/InfraMap.jsx`
- `frontend/src/features/map/kakaoLoader.js`
- `frontend/src/legacy/infra.jsx`
- `frontend/index.html`
- `frontend/.env.example`

주요 내용:

- `kakaoLoader.js`에서 카카오 JavaScript SDK를 동적으로 로드하도록 구성했다.
- 카카오맵 JavaScript 키는 코드에 직접 넣지 않고 `VITE_KAKAO_MAP_KEY` 환경변수로 읽는다.
- `InfraMap.jsx`에서 주소 기준 좌표 변환, 지도 생성, 주변 시설 검색, 마커 표시 흐름을 담당한다.
- 검색 필터는 대중교통, 병원/약국, 마트/편의점, 반려동물 관련 장소 중심으로 구성했다.
- 기존 `legacy/infra.jsx`에서는 `t.mapSrc === "mock"`일 때만 목업 지도를 쓰고, 그 외에는 새 `LiveInfraMap`을 렌더링하도록 연결했다.
- `index.html`에서 Vite 진입점은 `/src/main.jsx` 하나만 사용하도록 정리했다. 카카오 SDK는 HTML에 직접 넣지 않고 loader에서 관리한다.

로컬 환경변수 예시:

```properties
VITE_KAKAO_MAP_KEY=카카오_JavaScript_키
```

```text
http://localhost:5173
http://127.0.0.1:5173
```

### 1.2 카카오 OAuth 콜백 화면 추가

카카오 로그인 성공 후 백엔드가 프론트로 JWT를 전달할 수 있도록 `/oauth/callback` 진입점을 추가했다.

추가/수정한 파일:

- `frontend/src/features/member/oauth.js`
- `frontend/src/features/member/OAuthCallback.jsx`
- `frontend/src/shared/api/authFetch.js`
- `frontend/src/main.jsx`

주요 내용:

- `oauth.js`
  - 카카오 로그인 시작 URL을 `http://localhost:8080/oauth2/authorization/kakao`로 이동시키는 helper를 추가했다.
  - JWT 저장 키는 `zipt_access_token`으로 정리했다.
  - 로그인/로그아웃/토큰 조회 helper를 분리했다.
- `OAuthCallback.jsx`
  - 백엔드에서 전달한 `?token=` 값을 읽어 localStorage에 저장한다.
  - 저장 후 `/`로 이동한다.
  - OAuth 실패 시 `?error=&message=` 값을 화면에 보여주도록 수정했다.
- `authFetch.js`
  - API 요청 시 localStorage의 JWT를 `Authorization: Bearer ...` 헤더로 붙인다.
  - 401 응답이 오면 토큰을 제거하고 로그아웃 상태로 처리한다.
- `main.jsx`
  - `window.location.pathname === "/oauth/callback"`일 때는 레거시 앱 대신 `OAuthCallback`만 렌더링한다.

백엔드 origin 기본값:

```text
http://localhost:8080
```

필요하면 프론트 환경변수로 변경 가능하다.

```properties
VITE_BACKEND_ORIGIN=http://localhost:8080
```
