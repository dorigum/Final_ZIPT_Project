### 2. 용어 정리ZIP 데이터 선정 기준 수립

주택도시보증공사(HUG) 용어집과 부동산 관련 법령을 검토하고, 일반 임차인에게 필요한 용어를 우선 제공하기 위한 선정 기준을 정리했다.

선정 기준:

- 전·월세 계약 과정에서 실제로 자주 접하는 용어인지
- 잘못 이해할 경우 보증금 손실이나 권리 침해로 이어질 가능성이 있는지
- 계약 전, 계약 중, 계약 후 체크리스트와 연결할 수 있는지
- 등기부등본 또는 분석 리포트에 표시될 가능성이 높은지
- 사용자가 검색하거나 질문할 가능성이 높은지

카테고리 구성:

- 보증금 안전·위험 예방
- 계약·계약서
- 주거 형태·주택 구조
- 면적·가격·정책

### 3. 공공기관 기반 용어 데이터 정제

초기 11개 목업 데이터에서 시작해 HUG 용어집을 바탕으로 핵심 용어를 50개로 확장한 뒤, 공식 정의와 추가 용어가 포함된 최신 엑셀을 기준으로 총 80개까지 정제했다.

원본 데이터:

- `docs/부동산 핵심 용어 정리표_주택도시보증공사(HUG).xlsx`

최종 데이터 현황:

| 카테고리 | 용어 수 |
| --- | ---: |
| 보증금 안전·위험 예방 | 25개 |
| 계약·계약서 | 30개 |
| 주거 형태·주택 구조 | 13개 |
| 면적·가격·정책 | 12개 |
| 합계 | 80개 |

데이터 처리 원칙:

- 엑셀의 `설명`은 카드에 우선 표시되는 쉬운 설명으로 사용한다.
- 엑셀의 `표준 정의`가 채워진 항목은 공식 정의로 반영한다.
- `표준 정의`가 빈 기존 용어는 이전 정의를 삭제하지 않고 그대로 유지한다.
- 출처가 입력된 항목은 공식 정의와 함께 기관명을 표시한다.
- 출처가 비어 있는 경우 화면에 빈 출처 문구를 표시하지 않는다.
- 괄호 안의 명칭은 검색 별칭으로 등록한다.

용어 정제 사례:

- `복비(중개보수)`를 `중개보수(복비)`로 변경했다.
- 이전 명칭은 검색 별칭으로 유지해 `복비`, `중개보수`, `복비(중개보수)` 검색을 모두 지원한다.
- `등기부등본`에는 `등기사항전부증명서` 별칭을 추가했다.
- `선순위채권`에는 띄어쓴 표현인 `선순위 채권`을 별칭으로 추가했다.

구현 내용:

- 프론트 화면과 동일한 네 개 카테고리를 백엔드 enum에 반영했다.
- 용어명, 별칭, 쉬운 설명을 기준으로 검색할 수 있다.
- 용어별 공식 정의, 출처, 쉬운 설명, 행동 지침, 위험 수준을 응답한다.

API:

```http
GET /api/v1/terms
GET /api/v1/terms?query={검색어}
GET /api/v1/terms?category={카테고리}
GET /api/v1/terms/{slug}
```

테스트 보강:

- 전체 용어 수가 80개인지 확인
- 카테고리가 네 개 제공되는지 확인
- 검색 결과가 올바르게 필터링되는지 확인
- 상세 응답에 공식 정의와 출처가 포함되는지 확인
- 정제 전 명칭인 `복비`로도 `중개보수(복비)`를 찾을 수 있는지 확인
- 존재하지 않는 용어 요청 시 `TERM_NOT_FOUND`를 반환하는지 확인

### 5. 프론트 용어 정리ZIP 화면 연동

- 백엔드를 실행하지 않은 경우에도 동일한 80개 로컬 데이터를 fallback으로 표시한다.
- 프론트 fallback 데이터와 백엔드 시드 데이터의 내용을 완전히 동일하게 유지했다.
- 전체 및 카테고리별 용어 개수를 화면에 표시한다.
- 용어명, 별칭, 쉬운 설명을 이용한 검색을 제공한다.
- 카드를 펼치면 `공식·표준 정의`와 기관 출처를 확인할 수 있다.
- 기존에 작성한 임차인 행동 지침과 위험 배지는 해당 용어에 계속 표시한다.

### 8. 실행 시 유의사항

H2는 인메모리 데이터베이스이므로 백엔드를 재시작하면 최신 `terms.json` 시드가 다시 적재된다.

이미 백엔드가 실행 중인 상태에서 JSON만 변경하면 기존 프로세스는 이전 데이터를 계속 반환할 수 있다. 변경 후 아래 명령으로 백엔드를 재시작해야 한다.

```powershell
cd backend
.\gradlew.bat bootRun
```

프론트 화면에서 아래 두 표시가 보이면 최신 API가 정상 연결된 상태다.

```text
80개
API 연결됨
```

### 2. 용어 정리ZIP 데이터 선정 기준 수립

주택도시보증공사(HUG) 용어집과 부동산 관련 법령을 검토하고, 일반 임차인에게 필요한 용어를 우선 제공하기 위한 선정 기준을 정리했다.

선정 기준:

- 전·월세 계약 과정에서 실제로 자주 접하는 용어인지
- 잘못 이해할 경우 보증금 손실이나 권리 침해로 이어질 가능성이 있는지
- 계약 전, 계약 중, 계약 후 체크리스트와 연결할 수 있는지
- 등기부등본 또는 분석 리포트에 표시될 가능성이 높은지
- 사용자가 검색하거나 질문할 가능성이 높은지

카테고리 구성:

- 보증금 안전·위험 예방
- 계약·계약서
- 주거 형태·주택 구조
- 면적·가격·정책

### 3. 공공기관 기반 용어 데이터 정제

초기 11개 목업 데이터에서 시작해 HUG 용어집을 바탕으로 핵심 용어를 50개로 확장한 뒤, 공식 정의와 추가 용어가 포함된 최신 엑셀을 기준으로 총 80개까지 정제했다.

원본 데이터:

- `docs/부동산 핵심 용어 정리표_주택도시보증공사(HUG).xlsx`

최종 데이터 현황:

| 카테고리 | 용어 수 |
| --- | ---: |
| 보증금 안전·위험 예방 | 25개 |
| 계약·계약서 | 30개 |
| 주거 형태·주택 구조 | 13개 |
| 면적·가격·정책 | 12개 |
| 합계 | 80개 |

데이터 처리 원칙:

- 엑셀의 `설명`은 카드에 우선 표시되는 쉬운 설명으로 사용한다.
- 엑셀의 `표준 정의`가 채워진 항목은 공식 정의로 반영한다.
- `표준 정의`가 빈 기존 용어는 이전 정의를 삭제하지 않고 그대로 유지한다.
- 출처가 입력된 항목은 공식 정의와 함께 기관명을 표시한다.
- 출처가 비어 있는 경우 화면에 빈 출처 문구를 표시하지 않는다.
- 괄호 안의 명칭은 검색 별칭으로 등록한다.

용어 정제 사례:

- `복비(중개보수)`를 `중개보수(복비)`로 변경했다.
- 이전 명칭은 검색 별칭으로 유지해 `복비`, `중개보수`, `복비(중개보수)` 검색을 모두 지원한다.
- `등기부등본`에는 `등기사항전부증명서` 별칭을 추가했다.
- `선순위채권`에는 띄어쓴 표현인 `선순위 채권`을 별칭으로 추가했다.
