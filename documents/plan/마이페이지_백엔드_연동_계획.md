1. 마이페이지 실제 로그인 사용자 정보 백엔드 연동 계획 개요
   - 마이페이지 상단의 유저 프로필 헤더(이름, 소셜 로그인 타입, 관심사 태그, 잔여 무료 분석 횟수)에 실제 로그인한 사용자의 정보를 유기적으로 매핑하고 연동하기 위한 백엔드 API 연동 명세와 프론트엔드 연동 흐름을 기술한 마스터 계획서입니다.

2. 백엔드 사용자 정보 조회 API 규격 제안
   - HTTP Request: GET /api/member/me (Headers: Authorization: Bearer <accessToken>)
   - HTTP Response DTO:
     - email (String): 사용자 이메일
     - name (String): 사용자 이름/닉네임
     - provider (String): 가입 및 인증 수단 (LOCAL / KAKAO / NAVER / GOOGLE)
     - tags (List<String>): 가입 시 설정한 관심 태그 (예: ["임차인", "아파트 선호"])
     - remainingFreeCount (Integer): 이번 달 잔여 무료 분석 횟수
