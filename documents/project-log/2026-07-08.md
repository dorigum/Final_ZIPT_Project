### **🏆 엄선된 핵심 트러블슈팅 TOP 2**

#### **1️⃣ [백엔드/인프라] MySQL ENUM 타입 컬럼 불일치로 인한 AI 비동기 분석 중단 장애 해결**

> **태그**: `260706[fix] mysql processing_status enum 확장`

- **선정 이유**: 단순한 코드 오타 수정이 아닌, **"JPA DDL-auto: update 설정의 한계로 인한 DB-애플리케이션 간 런타임 불일치 장애"**를 추적하여 해결한 매우 값진 풀스택 문제 해결 사례입니다.
- **이슈 내용 요약**:
    - **현상**: 임대차계약서 이미지 업로드 시, AI 분석이 진행 단계를 넘기지 못하고 상세 원인 없이 즉각 `처리실패` 처리됨.
    - **원인**: 백엔드 자바 코드에 상세 분석 전이 단계(`OCR_EXTRACTING`, `AI_EXTRACTING` 등)가 새롭게 추가되었으나, 실제 Docker MySQL 데이터베이스의 `processing_status` 컬럼은 이전 규격인 `ENUM('COMPLETED', 'FAILED', 'PROCESSING')`에 머물러 있어 실시간 상태 갱신 쿼리 시 `Data truncated` 트랜잭션 오류가 발생하여 스레드가 뻗어버림.
    - **해결**: 컨테이너 DB에 직접 접속하여 수동으로 `ALTER TABLE contracts MODIFY COLUMN...` DDL을 실행해 컬럼 규격을 백엔드 엔티티의 최신 후보군과 완벽하게 동기화함.

#### **2️⃣ [프론트엔드] React Query 캐시 키 데이터타입(String vs Number) 엇갈림으로 인한 실시간 UI 미동기화 해결**

> **태그**: `260706[fix] useContract react query 캐시 키 numeric 캐스팅`

- **선정 이유**: 리액트 비동기 상태 관리 라이브러리인 **React Query의 캐시 동치 비교(`===`) 메커니즘을 정확히 파악**하고, 데이터 흐름상 묵시적 타입 변환 오류를 방어해 낸 정교한 상태 튜닝 사례입니다.
- **이슈 내용 요약**:
    - **현상**: 계약서 상세 분석 리포트 화면에서 알림 설정(D-day Tracking) 토글 스위치를 온/오프했을 때, 백엔드 DB에는 정상 저장되지만 마이페이지로 이동했을 때 알림 칩이 실시간으로 켜지거나 꺼지지 않고 F5 새로고침을 해야만 뒤늦게 반영되는 현상.
    - **원인**: `useParams()`로 넘겨받은 `contractId`는 문자열형(`"12"`)인 반면, 마이페이지 리포트 이력 및 React Query 내 캐시 데이터의 키는 숫자형(`12`)이어서 React Query가 캐시 무효화(`invalidateQueries`) 대상을 서로 다른 캐시로 인식하여 엇갈렸음.
    - **해결**: `useContract` 훅 내의 모든 `contractId` 파라미터를 `Number(contractId)` 로 명시적 강제 형변환(Casting)하여 캐시 키 정합성을 정수형으로 일관되게 일치시킴.
